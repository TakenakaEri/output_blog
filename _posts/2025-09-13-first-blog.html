今回、自分の学習記録用のアウトプットブログを作成しました。
作成方法は以下のURLを参考にさせていただきました。

ゼロからわかる！GitHub Pagesを使った自前無料ブログの作り方（Jekyll）

基本的に上記のブログを見ていただければ、爆速でサイトを公開することができると思います。

構築環境

  • OS: macOS
  • Ruby: 3.3.0
  • Jekyll: 4.4.1
  • ホスティング: GitHub Pages

今回は、Dockerは使用せずに、記事通りローカル環境でRubyとJekyll(ジキル)を使用しました。

実装の際にであったエラー

ローカルのURLの修正

記事の中で「localhost:4000にアクセスするとサイトが見れる」と記載があったのですが、アクセスしてみるとエラー画面が出ていました。

実行ログを見ると 


    Server address: http://127.0.0.1:4000/startbootstrap-clean-blog-jekyll/
とログが出ていたので、ログのURLにアクセスしてみるとサイトが表示されました。

これは、設定ファイルである_config.ymlのbaseurlに設定を行っているため、サブディレクトリの指定が必要なためでした。

_config.yml 

  title: Output Blog 
  email: takenakaeri0423@gmail.com 
  description: 日々のひとこと学習ブログ 
  author: Start 
  Bootstrap url: "https://github.com/TakenakaEri" 
  baseurl: "/startbootstrap-clean-blog-jekyll/"

サブディレクトリの設定についてきちんと理解できていなかったので、勉強になりました。

開発環境と本番環境の設定の追加

上記の_config.ymlに関してですが、デフォルトのままだと開発環境用で、本番環境ではCSSが当たりませんでした。

原因は、設定ファイルで本番環境用のURLが登録されていなかったことだったので、本番環境用と開発環境でbaseurlの設定を変更することにしました。


  # 本番用（GitHub Pages）
  baseurl: "/output_blog"
  url: "https://github.com/TakenakaEri"
  # 開発用
  # baseurl: "/startbootstrap-clean-blog-jekyll/"

GitHub Pagesでブログを公開する予定だったので、本番用のサブディレクトリは作成したリポジトリ名を指定しました。

ゆくゆくは、本番用と開発用を自動で切り替えれるように設定しようかと思いますが、一旦コメントアウトで対応しています。

今回の学び

Jekyllについて

GitHub上に公開されているテンプレートをZipファイル形式でダウンロードして使用したのが初めてだったので、良い経験になりました。

このブログでは、技術的なことを書いていきたいと思っているので、SCSSは変更しないと見づらいなと感じているため、少しずつ手を加えていきたいと思います。

なお、今回使用したテンプレートだと、ブログはHTML形式で記載するテンプレでしたが、Markdown形式で書けるほうがスピード感を持ってブログが作成できるのではないかなと思います。

(私は勉強がてらこのままHTML形式で書いていこうと思います笑)

※ なお、Jekyllのローカルサーバーでは、https非対応なようです。ローカルなので、私はそこまで気にせず使っていますが、気になる人は避けたほうがいいかもしれません。

※ また、今回HTMLのフォーマットを使用していますが、ブログを書く際にHTMLの記述が間違っているとページが表示されないので、注意！！

ローカル非対応の弊害

  • ブラウザAPIの制限(使えないAPIがある)
  • Service Workerが使えない(バックグラウンドで動くJavaScript)
  • Push通知が動作しない
  • セキュリティ機能の制限(Secure Cookie が設定できない)
  • 本番環境との差異
  • Mixed Contentエラー(HTTPSとHTTPの差異による問題を事前に検知できない)
  • PWA開発時(HTTPでは必須機能が動作しない)

なお、JekyllをHTTP化することはできるようなので、勉強がてらいつかやります←

GitHub Pagesについて

今回初めて使用したのですが、驚くほど早くWebページを公開できて、感動しました。

簡単でGitHubが使える人なら使いやすいので、簡単な用途のサイトであれば、GitHub Pagesで事足りると思います。
（勉強のためにサイトを作成するなら、サーバーとかDBとか自分で設定したほうがいいと思う）

改善したいこと

  • SCSSの修正（コードを自然に読めるように）
  • JekyllをHTTP化する
  • 設定ファイルを修正（本番用URLと開発用URLを自動で切り替えれるように設定）
  • ABOUT USなどの内容修正
